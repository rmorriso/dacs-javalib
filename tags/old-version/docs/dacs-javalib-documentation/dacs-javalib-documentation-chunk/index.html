Abstract

This is the system documentation for the DACS Java Library. Documentation on DACS - the Distributed Access Control System - may be found at the DACS home page. The DACS distribution itself is available for download from SourceForge: http://sourceforge.net/projects/dacs. This document is a beginning but readers should be aware that there are gaps in the coverage and that documentation lags behind code.

This first version of DACS Java Library implements Java wrapper classes for selected DACS services, and provides an HTTP client through which DACS services may be accessed and DACS credentials obtained and managed. The DACS Java Library is open source software and has been released as a companion project to DACS on SourceForge: http://sourceforge.net/projects/dacs-contrib

Note that the DACS Java Library adds no new functionality to DACS.

About DACS and the DACS Java Library

$Id: intro.xml 26 2005-10-17 16:43:43Z rmorriso $

DACS is a collection of software components that manages identities throughout a federation of distributed systems and enforces role-based access control, providing single sign-on functionality.

DACS is designed to be a general-purpose authentication and access control framework, suitable for a wide variety of environments and applications. DACS is commonly used in the context of web services, and in conjunction with the Apache web server, where web services are typically invoked by people using off-the-shelf web browsers as their user agent. DACS may be used to secure a single web server or as a full-fledged single sign-on and federated identity solution.

The DACS Java Library (DJL) is being developed to support the use of DACS in Java client applications. Support is provided in several key areas:

  • establishing the necessary preconditions for access

  • hiding implementation details of DACS Web services (HTTP request signatures, XML reply parsing, etc)

  • specialization of the excellent Jakarta Commons HttpClient library for DACS request processing

The DJL may be used in the development of Java applications deployed in thick-client (typically Java Swing) and middleware configurations as illustrated in Figure 1, “User Agent Configuration” and Figure 2, “Middleware Configuration” below.

Figure 2. Middleware Configuration

An example application of DJL in a user agent configuration is the User-friendly Desktop Internet GIS application, uDig, which uses the DJL to incorporate DACS authentication and notice acknowledgement that has been configured in front of Open Geospatial Consortium Web map servers. The DACS Federation Administration Web application, FedAdmin (also released on the SourceForge dacs-contrib project), is an example middleware application which uses the DJL not only for authentication but to invoke DACS web services for the administration of one or more DACS federations and their member jurisdictions.
